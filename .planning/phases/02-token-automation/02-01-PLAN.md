  <name>Task 1: 创建 OAuth PKCE 核心模块</name>
  <files>src/gallery_dl_auo/auth/oauth.py</files>
  <action>
创建 OAuth PKCE 核心实现模块,包含以下功能:

1. **PKCE 工具函数**:
   - `generate_pkce_verifier() -> str`: 生成 43-128 字符的 URL-safe 随机字符串
   - `generate_pkce_challenge(verifier: str) -> str`: 使用 SHA-256 + base64url 编码生成 challenge
   - 使用 `secrets.token_urlsafe(96)` 生成 verifier,截取到 128 字符
   - Challenge 生成时移除 base64 padding (`=`)

2. **OAuth 常量定义**:
   - `CLIENT_ID = "MOBrBDS8blbauoSck0ZfDbtuzpyT"`
   - `CLIENT_SECRET = "lsACyCD94FhDUtGTXi3QzcFE2uU1hqtDaKeqrdwj"`
   - `AUTH_URL = "https://app-api.pixiv.net/web/v1/login"`
   - `TOKEN_URL = "https://oauth.secure.pixiv.net/auth/token"`
   - 添加注释说明来源: Pixiv 官方客户端凭证

3. **错误处理**:
   - 定义 `OAuthError` 异常类 (继承自 Exception)
   - 包含错误消息和可选的 HTTP 状态码

实现规范:
- 导入: `secrets`, `hashlib`, `base64`
- 类型注解: 所有函数添加返回类型
- Docstring: 每个函数添加简洁的功能说明

**不要实现**: 浏览器自动化或 HTTP 服务器逻辑 (Task 2 负责)

<task type="auto">
  <name>Task 2: 实现 Pixiv OAuth 登录流程</name>
  <files>src/gallery_dl_auo/auth/pixiv_auth.py</files>
  <action>
创建 Pixiv OAuth 登录流程模块,实现完整的自动化登录:

1. **PixivOAuth 类**:
   ```python
   class PixivOAuth:
       def __init__(self):
           self.code_verifier = generate_pkce_verifier()
           self.code_challenge = generate_pkce_challenge(self.code_verifier)
   ```

2. **登录流程方法**:
   - `login() -> dict`: 完整登录流程,返回 {access_token, refresh_token}
     - 构造授权 URL (包含 code_challenge 和 code_challenge_method=S256)
     - 调用 `webbrowser.open(auth_url)` 打开系统默认浏览器
     - 显示友好的登录指导信息 (使用 Rich console)
     - 调用 `_wait_for_callback()` 等待用户完成登录
     - 调用 `_exchange_token(code)` 交换 token
     - 返回 token 对

3. **回调等待逻辑** `_wait_for_callback() -> str`:
   - **方案 A (主要)**: 使用 Playwright 监听导航事件
     - 启动浏览器上下文,监听 `page.on('framenavigated')` 事件
     - 捕获包含 `code=` 的 URL 并提取授权码
     - 设置无超时 (符合用户决策)
   - **方案 B (备选,如果方案 A 不可行)**: 使用 `input()` 手动输入
     - 提示用户从浏览器地址栏或开发者工具复制 code
     - 只在 Playwright 失败时使用

4. **Token 交换逻辑** `_exchange_token(code: str) -> dict`:
   - 使用 `requests.post(TOKEN_URL)` 交换 token
   - 请求体包含: client_id, client_secret, code, code_verifier, grant_type="authorization_code"
   - User-Agent: "PixivAndroidApp/5.0.64 (Android 6.0)"
   - 错误处理: 4xx/5xx 响应抛出 OAuthError
   - 返回 {access_token, refresh_token, expires_in}

5. **友好提示信息**:
   - 使用 Rich Console 打印彩色提示
   - 包含: "请在浏览器中完成登录", "程序将自动检测登录成功"
   - 如果使用手动输入方案: 提示如何获取 code

实现规范:
- 导入: `webbrowser`, `requests`, `rich.console.Console`
- 异常处理: 网络错误、OAuth 错误使用明确的错误消息
- 日志: 使用 `gallery_dl_auo` logger (继承自 Phase 1)
- **Playwright 依赖**: 如果使用 Playwright,添加到 pyproject.toml 的可选依赖

**注意**: Pixiv 不支持 localhost 回调,必须使用浏览器自动化或手动输入
