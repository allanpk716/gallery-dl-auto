**2. config/loader.py - 配置加载和验证**

实现配置加载和验证逻辑:

```python
from omegaconf import OmegaConf, DictConfig
from gallery_dl_auo.config.schema import AppConfig

def load_and_validate_config(cfg: DictConfig) -> AppConfig:
    """加载并验证配置

    Args:
        cfg: Hydra 加载的 DictConfig 对象

    Returns:
        AppConfig: 验证后的配置对象

    Raises:
        ValueError: 配置验证失败
    """
    # 转换为 dataclass 对象,触发类型验证
    config: AppConfig = OmegaConf.to_object(cfg)

    # 自定义验证逻辑
    if config.concurrent_downloads < 1:
        raise ValueError(
            "concurrent_downloads 必须大于 0\n"
            "建议值: 3-5(避免触发 pixiv 反爬虫)"
        )

    if config.request_interval < 0.5:
        raise ValueError(
            "request_interval 不能小于 0.5 秒\n"
            "建议值: 1.0-2.0 秒(避免触发 pixiv 反爬虫)"
        )

    if config.log_level.upper() not in ["DEBUG", "INFO", "WARNING", "ERROR", "CRITICAL"]:
        raise ValueError(
            f"log_level 必须是以下值之一: DEBUG, INFO, WARNING, ERROR, CRITICAL\n"
            f"当前值: {config.log_level}"
        )

    return config
```

**关键点**:
- 使用 dataclass 定义配置结构(类型安全)
- OmegaConf.to_object 自动类型转换和验证
- 自定义验证提供友好的错误提示
- 错误信息包含建议值

**避免**:
- 不要使用字典存储配置(无类型检查)
- 不要在验证中抛出通用异常(使用 ValueError 并提供解决建议)

**1. cli/main.py - 主命令组**

```python
import click
import logging
from rich.console import Console

from gallery_dl_auo import __version__
from gallery_dl_auo.utils.logging import setup_logging

@click.group()
@click.option("--verbose", "-v", is_flag=True, help="启用详细输出(DEBUG 级别)")
@click.option("--quiet", "-q", is_flag=True, help="静默模式(只显示 ERROR)")
@click.pass_context
def cli(ctx: click.Context, verbose: bool, quiet: bool) -> None:
    """Pixiv 排行榜下载器 - 自动化获取 token 并下载排行榜内容

    用户首次手动登录后,程序自动捕获、存储和更新 refresh token,
    无需手动从浏览器开发者工具中复制,实现真正的自动化下载流程。

    \b
    常用命令:
      pixiv-downloader version        显示版本信息
      pixiv-downloader config         查看当前配置
      pixiv-downloader doctor         诊断环境和配置
      pixiv-downloader download       下载排行榜内容(未来实现)
    """
    ctx.ensure_object(dict)

    # 确定日志级别
    log_level = "DEBUG" if verbose else "ERROR" if quiet else "INFO"
    ctx.obj["console"] = setup_logging(log_level)
    ctx.obj["verbose"] = verbose

    # 存储日志级别供子命令使用
    logger = logging.getLogger("gallery_dl_auo")
    ctx.obj["logger"] = logger

# 注册子命令(在文件末尾)
from gallery_dl_auo.cli.version import version
from gallery_dl_auo.cli.config_cmd import config_cmd
from gallery_dl_auo.cli.doctor import doctor

cli.add_command(version)
cli.add_command(config_cmd, name="config")
cli.add_command(doctor)

if __name__ == "__main__":
    cli()
```

**1. cli/config_cmd.py - config 子命令**

```python
import click
from rich.console import Console
from rich.table import Table
import yaml

@click.command()
@click.pass_context
def config_cmd(ctx: click.Context) -> None:
    """查看当前配置

    显示从 config.yaml 加载的配置值。
    """
    console: Console = ctx.obj["console"]

    try:
        # 加载配置文件
        with open("config.yaml", "r", encoding="utf-8") as f:
            config = yaml.safe_load(f)

        # 创建表格显示配置
        table = Table(title="当前配置", show_header=True, header_style="bold blue")
        table.add_column("配置项", style="cyan")
        table.add_column("值", style="green")

        for key, value in config.items():
            table.add_row(key, str(value))

        console.print(table)

    except FileNotFoundError:
        console.print("[bold red]错误:[/bold red] 找不到 config.yaml 文件")
        console.print("请确保 config.yaml 在当前目录下")
        raise click.Abort()
    except yaml.YAMLError as e:
        console.print(f"[bold red]错误:[/bold red] config.yaml 格式错误")
        console.print(f"详细信息: {e}")
        raise click.Abort()
```

**2. cli/doctor.py - doctor 子命令**

```python
import click
import sys
from rich.console import Console

@click.command()
@click.pass_context
def doctor(ctx: click.Context) -> None:
    """诊断环境和配置

    检查 Python 版本、依赖项和配置文件。
    """
    console: Console = ctx.obj["console"]

    console.print("[bold blue]运行诊断检查...[/bold blue]\n")

    # 检查 Python 版本
    python_version = sys.version_info
    if python_version >= (3, 10):
        console.print(f"✓ Python 版本: {python_version.major}.{python_version.minor}.{python_version.micro}")
    else:
        console.print(f"✗ Python 版本过低: {python_version.major}.{python_version.minor}.{python_version.micro}")
        console.print("  需要 Python 3.10 或更高版本")

    # 检查配置文件
    import os
    if os.path.exists("config.yaml"):
        console.print("✓ 配置文件: config.yaml")
    else:
        console.print("✗ 配置文件: config.yaml (不存在)")
        console.print("  提示: 程序将使用默认配置")

    # 检查依赖项
    dependencies = ["click", "hydra", "omegaconf", "rich"]
    for dep in dependencies:
        try:
            __import__(dep)
            console.print(f"✓ 依赖项: {dep}")
        except ImportError:
            console.print(f"✗ 依赖项: {dep} (未安装)")

    console.print("\n[bold green]诊断完成[/bold green]")
```

**关键点**:
- 使用 Rich Table 美化配置输出
- 错误处理提供清晰的提示
- doctor 命令检查 Python 版本、配置文件、依赖项
- 使用 ✓ 和 ✗ 符号表示状态

**避免**:
- 不要在 config 命令中修改配置(只读操作)
- 不要在 doctor 中执行耗时检查(快速诊断)
