---

## Objective

使用 Tenacity 库替换现有的简单 for 循环重试逻辑,实现生产级别的指数退避重试策略。确保所有网络请求和文件下载操作在失败时自动重试 3 次,使用指数退避(1秒→2秒→3秒),并通过日志记录重试过程。

---

## Context

**User Decision (LOCKED):**
- 网络请求失败时重试 3 次,使用指数退避策略(1秒、2秒、3秒)
- 错误消息友好且详细:显示错误类型 + 建议操作 + 原始错误信息
- 记录错误日志到文件,包含时间戳、错误类型、上下文信息

**Research Findings:**
- Tenacity 是 Python 最成熟的重试库,支持装饰器模式
- wait_exponential(multiplier=1, min=1, max=3) 实现精确的 1→2→3 秒退避
- before_sleep_log 钩子可记录每次重试的详细信息

**Current Implementation:**
- `retry_handler.py` 使用简单 for 循环 + 固定 5 秒延迟
- 不支持指数退避,无详细重试日志

<files>

</files>

<action>

2. 重写 `retry_handler.py`:
   - 移除现有的 `retry_on_failure()` 和 `retry_download_file()` 函数
   - 新增 `retry_on_network_error()` 装饰器:
     - 使用 `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=1, max=3), retry=retry_if_exception_type(RequestException))`
     - 使用 `before_sleep_log(logger, logging.WARNING)` 记录每次重试
   - 新增 `retry_on_file_error()` 装饰器:
     - 针对 `IOError`, `PermissionError`, `OSError` 重试
     - 同样的指数退避策略
   - 保留 `retry_download_file()` 函数,但内部使用装饰器而非 for 循环
