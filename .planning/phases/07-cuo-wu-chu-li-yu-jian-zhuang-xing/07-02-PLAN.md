## Objective

使用 SQLite 数据库替换现有的 JSON 进度文件,记录每个作品的下载状态。支持高效查询已下载内容、按排行榜过滤、断点续传检测。数据库文件存储在 `~/.gallery-dl-auto/downloads.db`,确保跨目录的统一历史追踪。

---

## Context

**User Decision (LOCKED):**
- 使用 SQLite 数据库维护下载记录(作品 ID、文件路径等)
- 程序支持增量下载,跳过已下载的内容

**Research Findings:**
- SQLite 内置 ACID 事务保证,无需额外配置
- WAL 模式提升并发性能: `PRAGMA journal_mode=WAL`
- 推荐数据库位置: `~/.gallery-dl-auto/downloads.db` (全局统一)

**Current Implementation:**
- `progress_manager.py` 使用 JSON 存储已下载 ID 列表
- 每个排行榜一个 `.progress.json` 文件
- 大型排行榜(月榜 500+)时文件体积大,查询慢

<action>
1. 在 `paths.py` 中添加:
   ```python
   def get_download_db_path() -> Path:
       """获取下载历史数据库路径"""
       return get_config_dir() / "downloads.db"
   ```

2. 创建 `download_tracker.py`:
   - `DownloadTracker` 类:
     - `__init__(self, db_path: Path)`: 初始化数据库连接,调用 `_init_database()`
     - `_init_database(self)`: 创建表和索引
       - 表 `downloads`: `illust_id INTEGER PRIMARY KEY, file_path TEXT, mode TEXT, date TEXT, downloaded_at TIMESTAMP, file_size INTEGER, checksum TEXT`
       - 索引 `idx_mode_date`: 加速按排行榜查询
       - 索引 `idx_file_path`: 检测重复文件
       - 执行 `PRAGMA journal_mode=WAL` 启用 WAL 模式
     - `is_downloaded(self, illust_id: int) -> bool`: 查询作品是否已下载
     - `record_download(self, illust_id: int, file_path: Path, mode: str, date: str, file_size: int | None = None)`: 记录下载完成
     - `get_pending_illusts(self, mode: str, date: str, all_illusts: list[int]) -> list[int]`: 获取待下载 ID(排除已下载)
     - `get_downloaded_count(self, mode: str, date: str) -> int`: 统计某排行榜已下载数量
     - `cleanup_failed_downloads(self, days: int = 7)`: 清理 7 天前的失败记录(可选,为未来准备)
   - 使用 `with sqlite3.connect(db_path) as conn` 确保连接自动关闭
   - 所有数据库操作使用参数化查询,避免 SQL 注入
</action>
