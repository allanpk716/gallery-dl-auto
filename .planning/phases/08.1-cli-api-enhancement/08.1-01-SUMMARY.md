# Phase 8.1 Plan 01: JSON 帮助系统

**Status:** Complete

**Duration:** ~15 minutes

## Summary

实现了 --json-help 全局参数,为第三方工具提供结构化的 CLI 元数据。创建了 json_help.py 模块,使用 Click 的 `Context.to_info_dict()` API 提取命令信息,返回包含命令名称、参数、类型的规范 JSON 格式。

## What Was Built

### 核心功能
1. **json_help.py 模块** - JSON 帮助生成器
   - `generate_json_help(cli_group)` - 生成结构化 JSON 帮助
   - `_map_param_type(param_type)` - 映射 Click 参数类型为简单类型
   - `_get_global_options()` - 返回全局选项元数据
   - 支持命令映射格式(对象键为命令名)
   - 合并全局选项到每个命令的参数列表

2. **主命令组扩展**
   - 添加 --json-help 全局参数
   - 使用 is_eager=True 确保在命令验证前执行
   - 使用 expose_value=False 隐藏参数传递
   - 回调函数 `_handle_json_help()` 输出 JSON 并退出

3. **测试覆盖**
   - `test_json_help_structure` - 验证 JSON 结构和命令存在
   - `test_json_help_completeness` - 验证参数字段完整性
   - `test_json_help_flag_works` - 验证 --json-help 参数工作
   - `test_json_help_contains_global_options` - 验证全局选项合并

### 技术决策
- **命令映射格式**: 使用对象键作为命令标识符,便于第三方工具直接通过命令名访问元数据
- **参数详细程度**: 完整描述包含 name, type, required, description, default_value
- **全局选项合并**: 避免第三方工具需要额外查询全局选项列表,简化集成复杂度
- **类型映射**: 使用基础类型字符串 ("string", "integer", "boolean", "array")
- **中文支持**: 使用 `ensure_ascii=False` 正确显示中文描述

## Verification

### CLI-01 需求验证

✅ **用户可以通过 --json-help 参数获取所有命令的结构化 JSON 帮助信息**

```bash
$ pixiv-downloader --json-help | jq .
{
  "download": {
    "name": "download",
    "description": "Download Pixiv ranking images",
    "parameters": [...]
  },
  "version": {...},
  "config": {...},
  ...
}
```

✅ **JSON 帮助包含所有命令的元数据(名称、参数、描述)**
- 所有 8 个命令都出现在 JSON 输出中
- 每个命令对象包含 name, description, parameters 字段

✅ **每个命令的参数列表包含全局选项**
- verbose, quiet, json_output, json_help 出现在所有命令的参数列表中

✅ **JSON 输出格式符合 CONTEXT.md 规范**
- 命令映射格式(对象键为命令名)
- 参数对象包含完整字段
- 类型使用基础字符串

✅ **第三方工具可以可靠解析 JSON 输出**
- jq 可以正确解析输出
- Python json 模块可以正确加载输出
- 中文描述正确显示(非 Unicode 转义)
