## Summary

本阶段为 gallery-dl-auto 添加第三方 CLI 集成所需的 API 功能。核心任务是在现有 Click 8.3.1 CLI 框架上实现三个全局参数:`--json-help`(结构化帮助信息)、`--quiet`(静默模式)和 `--json-output`(统一 JSON 输出),确保所有命令输出在 JSON 模式下可解析。

项目已经具备良好的基础:使用 Click 框架构建 CLI、Pydantic 模型进行 JSON 序列化、标准化错误码系统和结构化日志。本阶段主要工作是扩展全局参数系统并实现 JSON 元数据提取。

**Primary recommendation:** 使用 Click 的 `@click.group()` 装饰器添加全局参数,利用 `ctx.to_info_dict()` 进行命令元数据提取,通过全局上下文对象传递模式标志,在命令执行前拦截并处理特殊模式。

## User Constraints (from CONTEXT.md)

### Locked Decisions

#### JSON 帮助结构
- **顶层结构**: 使用命令映射格式,而非命令数组
  ```json
  {
    "download": {...},
    "auth": {...}
  }
  ```
- **命令元数据**: 每个命令对象包含基础字段 `name`, `description`, `parameters`
- **参数详细程度**: 参数对象使用完整描述,包含 `name`, `type`, `required`, `description`, `default_value`
- **全局选项处理**: 全局选项(如 `--json-help`, `--quiet`)合并到每个命令的参数列表中,便于第三方工具直接使用

#### 错误响应格式
- **统一结构**: 成功和失败使用相同的顶层结构
  ```json
  {
    "success": false,
    "error": {
      "error_code": "auth_failed",
      "message": "...",
      "details": {...}
    },
    "data": null
  }
  ```
- **错误字段**: 包含基础字段 `error_code`, `message`, `details`(可选)
- **错误码格式**: 使用字符串标识符(如 `auth_failed`, `rate_limit_exceeded`),而非数字编码
- **部分结果处理**: 批量操作部分失败时,响应中仅包含错误信息,不包含已成功的数据

#### 模式交互规则
- **优先级**: `--json-output` > `--quiet` > `--verbose`,高优先级覆盖低优先级
- **`--quiet` 行为**: 禁用所有进度和日志输出,仅输出最终 JSON 结果,不保留错误输出(stderr)
- **`--json-output` vs `--quiet`**:
  - `--json-output`: 确保所有输出都是有效 JSON,包括错误信息也能被 JSON 解析器正确解析
  - `--quiet`: 仅静默进度但不保证 JSON 格式,用于简洁输出
  - 两者行为不同,`--json-output` 是 API 集成专用,`--quiet` 是通用简洁模式
- **`--verbose` 组合行为**: 与 `--quiet` 或 `--json-output` 冲突时被忽略,不添加额外字段

#### 元数据完整性
- **默认值**: 参数对象中总是包含 `default_value` 字段,即使参数是必需的(此时为 `null`)
- **类型信息**: 使用基础类型字符串 `"string"`, `"integer"`, `"boolean"`, `"array"`,不包含枚举值或范围限制
- **Required 标记**: 使用 `required` 布尔字段明确标记参数是否必需,不通过 `default_value` 推断
- **描述语言**: 所有描述和帮助文本使用中文,与项目主语言一致

### Claude's Discretion
- JSON 帮助的具体嵌套层级和字段顺序
- `details` 字段在错误响应中的具体结构(根据错误类型动态调整)
- 参数类型为 `array` 时的元素类型说明
- `default_value` 的格式表示(如数组默认值用 `[]` 还是 `null`)

### Pattern 1: 全局参数注册和传递
**What:** 在主命令组添加全局参数,通过 Click Context 传递给所有子命令
**When to use:** 所有需要全局行为的参数,如 `--verbose`, `--quiet`, `--json-output`
**Example:**
```python
# Source: Context7 /pallets/click - Command Group with Global Options
import click

@click.group()
@click.option("--verbose", "-v", is_flag=True, help="详细模式")
@click.option("--quiet", "-q", is_flag=True, help="静默模式")
@click.option("--json-output", is_flag=True, help="JSON 输出模式")
@click.pass_context
def cli(ctx: click.Context, verbose: bool, quiet: bool, json_output: bool) -> None:
    """Pixiv 排行榜下载器"""
    ctx.ensure_object(dict)

    # 优先级处理: json_output > quiet > verbose
    if json_output:
        ctx.obj["output_mode"] = "json"
    elif quiet:
        ctx.obj["output_mode"] = "quiet"
    else:
        ctx.obj["output_mode"] = "normal"

    ctx.obj["verbose"] = verbose and not (quiet or json_output)

    # 配置日志系统
    setup_logging(
        log_level="DEBUG" if ctx.obj["verbose"] else "INFO",
        verbose=ctx.obj["verbose"],
        quiet=(ctx.obj["output_mode"] in ["quiet", "json"])
    )
```

### Pattern 2: JSON 帮助元数据提取
**What:** 使用 Click 的 `Context.to_info_dict()` 提取命令结构,转换为用户定义的 JSON 格式
**When to use:** 实现 `--json-help` 参数时
**Example:**
```python
# Source: Context7 /pallets/click - Command Introspection
import click
import json

def generate_json_help(cli_group: click.Group) -> dict:
    """生成结构化 JSON 帮助信息

    Args:
        cli_group: Click 主命令组

    Returns:
        命令映射字典,格式见 CONTEXT.md
    """
    commands_metadata = {}

    # 遍历所有注册命令
    for cmd_name, cmd in cli_group.commands.items():
        # 创建临时上下文以提取元数据
        with cmd.make_context(cmd_name, [], parent=None) as ctx:
            info = ctx.to_info_dict()

            # 提取参数信息
            parameters = []
            for param in cmd.params:
                param_info = {
                    "name": param.name,
                    "type": _get_param_type(param),
                    "required": param.required if hasattr(param, 'required') else False,
                    "description": param.help or "",
                    "default_value": param.default
                }
                parameters.append(param_info)

            # 添加全局选项到每个命令
            global_options = _get_global_options()
            parameters.extend(global_options)

            commands_metadata[cmd_name] = {
                "name": cmd_name,
                "description": cmd.help or "",
                "parameters": parameters
            }

    return commands_metadata

def _get_param_type(param: click.Parameter) -> str:
    """将 Click 参数类型映射为简单类型字符串"""
    type_mapping = {
        click.types.StringParamType: "string",
        click.types.IntParamType: "integer",
        click.types.BoolParamType: "boolean",
        click.types.Path: "string",
    }
    for click_type, simple_type in type_mapping.items():
        if isinstance(param.type, click_type):
            return simple_type
    return "string"  # 默认类型
```

### Pattern 3: 静默模式和 JSON 输出拦截
**What:** 在命令执行前检查模式标志,静默模式时重定向输出流,JSON 模式时捕获异常
**When to use:** 实现 `--quiet` 和 `--json-output` 时
**Example:**
```python
# Source: WebSearch - Click CLI quiet mode implementation
import sys
import os
from contextlib import redirect_stdout, redirect_stderr

@click.pass_context
def execute_with_mode(ctx: click.Context, command_func):
    """根据模式执行命令

    Args:
        ctx: Click 上下文
        command_func: 实际命令函数
    """
    output_mode = ctx.obj.get("output_mode", "normal")

    if output_mode == "json":
        # JSON 模式:捕获所有输出和异常
        try:
            with open(os.devnull, 'w') as devnull:
                with redirect_stdout(devnull), redirect_stderr(devnull):
                    result = command_func()
            # 仅输出最终 JSON 结果
            print(result)
        except Exception as e:
            # 将异常转换为 JSON 错误响应
            error_output = {
                "success": False,
                "error": {
                    "error_code": "internal_error",
                    "message": str(e),
                    "details": {"exception_type": type(e).__name__}
                },
                "data": None
            }
            print(json.dumps(error_output, ensure_ascii=False, indent=2))

    elif output_mode == "quiet":
        # 静默模式:仅禁用 stdout/stderr,保留最终结果
        with open(os.devnull, 'w') as devnull:
            with redirect_stdout(devnull), redirect_stderr(devnull):
                result = command_func()
        print(result)

    else:
        # 正常模式:标准执行
        return command_func()
```

### Anti-Patterns to Avoid
- **不要在子命令中重复检查全局参数**: 使用 Click Context 传递全局状态,避免每个命令都检查 `--quiet` 标志
- **不要直接修改 sys.stdout/stderr**: 使用 `contextlib.redirect_stdout` 临时重定向,避免影响其他模块
- **不要忽略异常的 JSON 序列化**: `--json-output` 模式下,所有异常都必须转换为 JSON 格式,包括系统异常
- **不要在 JSON 帮助中硬编码命令信息**: 使用 Click 的元数据提取功能,确保与实际命令定义保持一致

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| 命令元数据提取 | 手动遍历命令参数和帮助文本 | `Context.to_info_dict()` | Click 8.0+ 提供标准 API,自动处理所有参数类型和嵌套结构 |
| JSON 序列化 | 自定义 JSON encoder | `Pydantic.model_dump_json()` | 已有模型支持,自动处理枚举、Optional 和嵌套对象 |
| 输出流重定向 | 手动修改 sys.stdout | `contextlib.redirect_stdout` | Python 标准库,正确处理上下文管理器和异常 |
| 错误码管理 | 字符串常量或数字编码 | `utils/error_codes.py` 的 `ErrorCode` 枚举 | 已有标准化错误码系统,确保一致性 |

**Key insight:** Click 8.0+ 已经提供了命令元数据提取能力,不需要手动解析帮助文本或遍历参数对象。Pydantic 的 JSON 序列化已经过生产验证,可以正确处理复杂类型。

## Common Pitfalls

### Pitfall 1: 全局参数和子命令参数冲突
**What goes wrong:** 子命令中定义了与全局参数同名的参数,导致 Click 报错或参数被覆盖
**Why it happens:** Click 的参数解析是分层的,全局参数和子命令参数在不同层级
**How to avoid:**
- 使用 `ctx.obj` 存储全局状态,而非函数参数
- 在主命令组中使用 `expose_value=False` 隐藏全局参数,仅存储到上下文
```python
@click.option("--quiet", is_flag=True, expose_value=False, callback=lambda ctx, param, value: ctx.obj.setdefault("quiet", value))
```
**Warning signs:** 子命令函数签名包含全局参数名,Click 报错 "no such option"

### Pitfall 2: JSON 帮助缺少全局选项
**What goes wrong:** `--json-help` 输出的每个命令参数列表中缺少全局选项(如 `--verbose`, `--quiet`),第三方工具无法知道这些参数可用
**Why it happens:** Click 的 `to_info_dict()` 仅返回当前命令的参数,不包括父命令组的全局参数
**How to avoid:**
- 在生成 JSON 帮助时,显式添加全局选项到每个命令的参数列表
- 定义一个 `_get_global_options()` 函数返回全局参数元数据,合并到每个命令的 parameters 中
**Warning signs:** JSON 帮助输出中某个命令的参数列表与实际可用参数不符

### Pitfall 3: 静默模式未禁用日志输出
**What goes wrong:** `--quiet` 模式下仍有进度条或日志输出到 stderr,污染第三方工具的输出捕获
**Why it happens:** 日志系统配置在主命令组初始化时,但 `--quiet` 标志未正确传递给日志系统
**How to avoid:**
- 修改 `setup_logging()` 添加 `quiet` 参数,禁用所有 handler
- 在 `--quiet` 和 `--json-output` 模式下,不添加任何控制台 handler
**Warning signs:** 测试中 `runner.invoke(cli, ["--quiet", "download"])` 的 stderr 输出非空

### Pitfall 4: JSON 模式下的异常未被捕获
**What goes wrong:** `--json-output` 模式下,未捕获的异常导致程序崩溃并输出 Python traceback,而非 JSON 错误响应
**Why it happens:** 异常处理仅在命令函数内部,未覆盖 Click 的命令调度和参数验证阶段
**How to avoid:**
- 使用 Click 的 `result_callback` 或自定义 `main()` 函数包装整个命令执行
- 在最外层捕获所有异常,转换为 JSON 错误响应
```python
def main():
    try:
        cli()
    except Exception as e:
        if os.environ.get("GALLERY_DL_AUO_JSON_MODE"):
            print(json.dumps({"success": False, "error": {...}}))
        else:
            raise
```
**Warning signs:** `--json-output` 模式下,无效参数导致 Click 输出错误信息而非 JSON

### Pitfall 5: 中文描述在 JSON 中乱码
**What goes wrong:** JSON 输出中的中文帮助文本和错误消息显示为乱码或 Unicode 转义序列
**Why it happens:** JSON 序列化时未设置 `ensure_ascii=False`,或 Click 帮助文本编码问题
**How to avoid:**
- 所有 JSON 输出使用 `json.dumps(..., ensure_ascii=False)`
- Pydantic 模型序列化使用 `model_dump_json(..., ensure_ascii=False)`
- 确保源文件编码为 UTF-8
**Warning signs:** JSON 输出中包含 `\u4e2d\u6587` 这样的转义序列

### 提取命令元数据生成 JSON 帮助
```python
# Source: Context7 /pallets/click - to_info_dict() method
import click
import json

def generate_json_help(cli_group: click.Group) -> dict:
    """生成结构化 JSON 帮助信息"""
    commands_metadata = {}

    for cmd_name, cmd in cli_group.commands.items():
        # 跳过隐藏命令
        if getattr(cmd, "hidden", False):
            continue

        # 提取参数信息
        parameters = []
        for param in cmd.params:
            param_info = {
                "name": param.name,
                "type": _map_param_type(param.type),
                "required": getattr(param, "required", False),
                "description": param.help or "",
                "default_value": param.default if param.default is not click.core.UNSET else None
            }
            parameters.append(param_info)

        # 添加全局选项
        global_options = [
            {
                "name": "verbose",
                "type": "boolean",
                "required": False,
                "description": "详细模式:显示调试信息",
                "default_value": False
            },
            {
                "name": "quiet",
                "type": "boolean",
                "required": False,
                "description": "静默模式:禁用所有输出",
                "default_value": False
            },
            {
                "name": "json_output",
                "type": "boolean",
                "required": False,
                "description": "JSON 输出模式:所有输出为 JSON 格式",
                "default_value": False
            }
        ]
        parameters.extend(global_options)

        commands_metadata[cmd_name] = {
            "name": cmd_name,
            "description": cmd.help.strip().split('\n')[0] if cmd.help else "",
            "parameters": parameters
        }

    return commands_metadata

def _map_param_type(param_type: click.types.ParamType) -> str:
    """映射 Click 参数类型到简单类型字符串"""
    if isinstance(param_type, click.types.IntParamType):
        return "integer"
    elif isinstance(param_type, click.types.BoolParamType):
        return "boolean"
    elif isinstance(param_type, click.types.FloatParamType):
        return "number"
    elif isinstance(param_type, (click.types.StringParamType, click.types.Path)):
        return "string"
    elif isinstance(param_type, click.types.Tuple):
        return "array"
    else:
        return "string"
```

```python
    output_mode = ctx.obj.get("output_mode", "normal")

    if output_mode == "json":
        # JSON 模式:禁用所有输出,仅输出最终 JSON
        with open(os.devnull, 'w') as devnull:
            with redirect_stdout(devnull), redirect_stderr(devnull):
                try:
                    # 执行实际命令逻辑
                    result = _execute_actual_command(ctx)
                except Exception as e:
                    # 转换异常为 JSON
                    result = {
                        "success": False,
                        "error": {
                            "error_code": "internal_error",
                            "message": str(e),
                            "details": {"exception_type": type(e).__name__}
                        },
                        "data": None
                    }

        # 输出最终 JSON 结果
        if isinstance(result, dict):
            print(json.dumps(result, ensure_ascii=False, indent=2))
        else:
            print(result)  # 假设已经是 JSON 字符串(Pydantic 模型)

    elif output_mode == "quiet":
        # 静默模式:禁用进度和日志,保留最终结果
        with open(os.devnull, 'w') as devnull:
            with redirect_stdout(devnull), redirect_stderr(devnull):
                result = _execute_actual_command(ctx)
        print(result)

    else:
        # 正常模式:标准执行
        return _execute_actual_command(ctx)
```

## Open Questions

1. **`--json-help` 的执行时机**
   - What we know: `--json-help` 需要在命令实际执行前输出帮助并退出
   - What's unclear: 是否应该使用 `is_eager=True` 确保 `--json-help` 在参数验证前执行,还是应该在主命令组回调中处理
   - Recommendation: 使用 `is_eager=True` 和 `expose_value=False`,确保 `--json-help` 不影响正常命令执行流程,参考 Click 的 `--help` 实现

2. **错误响应的 `details` 字段格式**
   - What we know: `details` 字段应该包含错误的额外信息
   - What's unclear: 不同错误类型的 `details` 结构应该如何定义(固定字段还是动态字段)
   - Recommendation: 使用动态字段,根据 `error_code` 决定 `details` 内容。例如:
     - `AUTH_TOKEN_EXPIRED`: `{"expires_at": "2026-02-25T12:00:00Z"}`
     - `DOWNLOAD_FAILED`: `{"illust_id": 12345, "url": "https://..."}`
     - 保持灵活性,在实现时根据实际错误类型定义

3. **全局选项的类型信息**
   - What we know: 全局选项应该出现在每个命令的参数列表中
   - What's unclear: 全局选项的 `default_value` 应该反映全局默认值还是当前命令的默认值
   - Recommendation: 使用全局默认值(如 `--verbose` 的 `default_value` 为 `False`),因为这是参数的"默认"状态,而非实际运行时的状态
