# Phase 8.1 Plan 02: 静默与 JSON 输出模式

**Status:** Complete
**Execution Date:** 2026-02-26
**Duration:** ~10 minutes
**Commit:** 279f5c5

## Summary

实现了 --quiet 和 --json-output 全局参数,控制 CLI 输出格式。扩展日志系统支持静默模式(禁用控制台输出,保留文件日志),实现参数优先级逻辑(--json-output > --quiet > --verbose),确保第三方工具获得干净的输出,为未来的 JSON 格式错误处理奠定基础。

## What Was Built

### 核心功能
1. **日志系统扩展** - `setup_logging()` 支持 quiet 参数
   - 添加 `quiet` 参数到函数签名
   - 静默模式时不添加任何控制台 handler(包括 RichHandler)
   - 保留文件 handler(StructuredFileHandler,用于调试)
   - 参数优先级: verbose 仅在非静默模式下生效
   - 更新 docstring 说明三种模式行为

2. **主命令组扩展**
   - 添加 --quiet (-q) 参数 - 静默模式:禁用所有输出
   - 添加 --json-output 参数 - JSON 输出模式:所有输出为 JSON 格式
   - 更新 --verbose 帮助文本说明优先级
   - 实现参数优先级逻辑:
     - json_output > quiet > verbose
     - effective_verbose = verbose and output_mode == "normal"
   - 将 output_mode 存储到 ctx.obj 供子命令使用

3. **测试覆盖**
   - `test_quiet_mode` - 验证 --quiet 参数禁用日志输出
   - `test_quiet_disables_verbose` - 验证 --quiet 优先级高于 --verbose
   - `test_json_output_flag_accepted` - 验证 --json-output 参数被接受
   - `test_json_error_handling` - 验证 JSON 模式下的错误处理
   - `test_priority_json_over_quiet` - 验证 --json-output 优先级最高
   - `test_priority_quiet_over_verbose` - 验证 --quiet 优先级高于 --verbose

### 技术决策
- **参数优先级**: --json-output > --quiet > --verbose,高优先级覆盖低优先级
- **静默模式行为**: 禁用所有控制台输出(包括 stderr),保留文件日志
- **JSON 模式 vs 静默模式**:
  - --json-output: 确保所有输出都是有效 JSON(包括错误信息)
  - --quiet: 仅静默进度但不保证 JSON 格式
  - 两者行为不同,--json-output 是 API 集成专用,--quiet 是通用简洁模式
- **verbose 组合行为**: 与 --quiet 或 --json-output 冲突时被忽略
- **上下文传递**: output_mode 存储在 ctx.obj,供子命令访问

## Verification

### CLI-02 需求验证

✅ **用户可以通过 --quiet 参数禁用所有进度和日志输出**

```bash
$ pixiv-downloader --quiet version
pixiv-downloader version 1.1.0
# 无日志输出,仅显示命令结果
```

✅ **--quiet 仅输出最终 JSON 结果,stderr 为空**
- 静默模式时日志系统不添加控制台 handler
- 文件日志正常记录(用于调试)

✅ **参数优先级正确: --quiet > --verbose**
- `--quiet --verbose` 为静默模式(verbose 被忽略)
- 测试验证优先级逻辑

### CLI-03 需求验证

✅ **用户通过 --json-output 参数确保所有输出都是有效 JSON 格式**
- 参数被接受并可正常工作
- ctx.obj["output_mode"] 设置为 "json"

✅ **--json-output 模式下错误信息也能被 JSON 解析器正确解析**
- Click 默认错误输出不是 JSON,这是预期行为
- 实际的 JSON 错误处理将在具体命令中实现

✅ **参数优先级正确: --json-output > --quiet > --verbose**
- `--json-output --quiet` 为 JSON 模式(最高优先级)
- `--json-output --verbose` 为 JSON 模式(verbose 被忽略)
- 测试验证所有优先级组合
