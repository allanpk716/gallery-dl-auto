# Phase 02.1: 独立的 token 刷新命令 - Research

**Researched:** 2026-02-24
**Domain:** CLI 命令实现、JSON 输出格式、错误处理
**Confidence:** HIGH

## Summary

Phase 02.1 需要实现一个独立的 `pixiv-downloader refresh` 命令,用于手动或定时刷新 Pixiv token。这个命令将专注于 token 刷新功能,复用现有的 `PixivOAuth.validate_refresh_token()` 和 `TokenStorage` 架构。

现有代码库已经具备完整的 token 刷新基础设施:
- `PixivOAuth.validate_refresh_token()` - 验证并刷新 token
- `TokenStorage.save_token()` - 加密保存 token
- CLI 命令模式 (使用 Click 框架)

主要研究重点是:
1. 如何设计符合 CONTEXT.md 要求的 JSON 输出结构
2. 如何复用现有代码实现 token 刷新逻辑
3. 如何实现符合 Unix 标准的退出码
4. 如何实现 token 部分显示 (隐私保护)

**Primary recommendation:** 基于 `status_cmd.py` 的模式,创建新的 `refresh_cmd.py`,复用 `PixivOAuth.validate_refresh_token()` 和 `TokenStorage`,输出纯 JSON 格式,遵循 Unix 标准退出码。

## User Constraints (from CONTEXT.md)

### Implementation Decisions

#### 命令接口设计
- **命令名称:** `pixiv-downloader refresh`
- **参数:** 无必需参数,保持简单
- **可选参数:**
  - `--help/-h`: 显示帮助信息,包括所有参数功能描述和返回信息描述
- **输出格式:** 默认 JSON 格式输出 (方便第三方程序解析)
- **Token 显示:** 显示部分 token (如: abc...xyz),保护安全性的同时提供足够信息

#### JSON 输出结构
```json
{
  "status": "success" | "failed",
  "expires_at": "2026-02-24 20:00:00",
  "token": "abc...xyz",
  "error": "错误描述(仅在失败时)"
}
```

**字段说明:**
- `status`: 刷新结果状态
- `expires_at`: token 过期时间
- `token`: 新 token 的部分内容(前缀...后缀)
- `error`: 失败时的错误详情

#### 刷新行为
- **刷新前验证:** 总是直接刷新,不验证当前 token 有效性
- **自动保存:** 刷新成功后自动保存新 token 到加密存储
- **失败处理:** 刷新失败时明确提示用户运行 `pixiv-downloader login` 重新登录

#### 错误处理
- **Token 不存在:** 提示运行 `pixiv-downloader login` 命令
- **网络错误:** 立即失败,不自动重试 (让第三方程序决定重试策略)
- **退出码:**
  - `0`: 刷新成功
  - `1`: 刷新失败 (包括 token 不存在、网络错误、刷新失败等所有错误情况)

#### Claude's Discretion
- JSON 格式化的具体实现
- Token 部分显示的具体格式 (显示多少字符)
- 错误消息的详细程度和格式
- 日志记录级别

### Anti-Patterns to Avoid
- **使用 Rich Console 输出 JSON:** Rich 会添加控制字符,破坏 JSON 格式,导致第三方程序无法解析
- **在 JSON 输出中混合日志:** 应该将日志输出到 stderr,JSON 输出到 stdout
- **过于复杂的错误分类:** CONTEXT.md 要求所有错误都返回退出码 1,无需细分

## Common Pitfalls

### Pitfall 1: JSON 输出被 Rich 污染
**What goes wrong:** 使用 `console.print()` 输出 JSON 会导致 ANSI 转义序列混入,第三方程序解析失败
**Why it happens:** Rich 会自动添加颜色和格式化代码
**How to avoid:** 直接使用 `print()` 或 `sys.stdout.write()` 输出 JSON
**Warning signs:** JSON 输出中包含 `[`、`\x1b[` 等 ANSI 转义序列

### Pitfall 2: Token 显示泄露完整内容
**What goes wrong:** 在 JSON 输出中显示完整 token,违反隐私保护要求
**Why it happens:** 直接使用 `token_data["refresh_token"]` 而未进行遮蔽
**How to avoid:** 实现 token mask 函数,显示 `前缀...后缀` 格式
**Warning signs:** JSON 输出包含完整 refresh token

### Pitfall 3: 退出码不一致
**What goes wrong:** 不同错误返回不同退出码,与 CONTEXT.md 要求不符
**Why it happens:** 过度设计错误分类
**How to avoid:** 只使用 `sys.exit(0)` 和 `sys.exit(1)`,所有错误统一返回 1
**Warning signs:** 代码中包含 `sys.exit(2)`, `sys.exit(3)` 等

### Pitfall 4: 过期的 expires_at 计算
**What goes wrong:** 使用错误的时间计算导致 expires_at 不准确
**Why it happens:** 未考虑时区或使用了错误的时间基准
**How to avoid:** 使用 `datetime.datetime.now() + timedelta(seconds=expires_in)`,与 status_cmd.py 保持一致
**Warning signs:** expires_at 与实际过期时间不符

### Pitfall 5: 日志输出污染 JSON
**What goes wrong:** 日志和 JSON 输出混在 stdout,导致第三方程序解析失败
**Why it happens:** 使用 `print()` 输出日志
**How to avoid:** 日志使用 `logger` 输出到 stderr,JSON 使用 `print()` 输出到 stdout
**Warning signs:** 第三方程序报告 JSON 解析错误

## Code Examples

### Token Mask 函数
```python
def mask_token(token: str, prefix_len: int = 10, suffix_len: int = 10) -> str:
    """
    遮蔽 token 中间部分,只显示前缀和后缀

    Args:
        token: 完整 token
        prefix_len: 前缀长度 (默认 10)
        suffix_len: 后缀长度 (默认 10)

    Returns:
        遮蔽后的 token (如: "abc...xyz")
    """
    if len(token) <= prefix_len + suffix_len:
        # Token 太短,显示更少
        return token[:5] + "..." + token[-5:] if len(token) > 10 else token

    return token[:prefix_len] + "..." + token[-suffix_len:]
```

### JSON 输出模式
```python
import json
import sys
import datetime

def output_refresh_result(
    success: bool,
    token: str | None = None,
    expires_in: int | None = None,
    error: str | None = None
) -> None:
    """
    输出 JSON 格式的刷新结果

    Args:
        success: 刷新是否成功
        token: 新的 refresh token (成功时)
        expires_in: 过期时间(秒) (成功时)
        error: 错误信息 (失败时)
    """
    if success:
        # 计算过期时间
        expires_at = datetime.datetime.now() + datetime.timedelta(seconds=expires_in)
        output = {
            "status": "success",
            "expires_at": expires_at.strftime("%Y-%m-%d %H:%M:%S"),
            "token": mask_token(token),
        }
    else:
        output = {
            "status": "failed",
            "error": error,
        }

    # 输出 JSON 到 stdout
    print(json.dumps(output, indent=2, ensure_ascii=False))
```

### Refresh 命令骨架
```python
import sys
import click
from gallery_dl_auo.auth.pixiv_auth import PixivOAuth
from gallery_dl_auo.auth.token_storage import get_default_token_storage

@click.command()
def refresh() -> None:
    """Refresh Pixiv token

    Refreshes the stored refresh token and saves the new token.
    Outputs result in JSON format for third-party integration.
    """
    storage = get_default_token_storage()

    # 1. 加载 token
    token_data = storage.load_token()
    if not token_data:
        output_refresh_result(False, error="No token found. Run 'pixiv-downloader login' first.")
        sys.exit(1)

    refresh_token = token_data.get("refresh_token")
    if not refresh_token:
        output_refresh_result(False, error="Invalid token data (missing refresh_token).")
        sys.exit(1)

    # 2. 刷新 token (直接刷新,不预验证)
    result = PixivOAuth.validate_refresh_token(refresh_token)

    # 3. 处理结果
    if result["valid"]:
        # 保存新 token
        storage.save_token(
            refresh_token=result["refresh_token"],
            access_token=result.get("access_token"),
        )

        # 输出成功 JSON
        output_refresh_result(
            True,
            token=result["refresh_token"],
            expires_in=result.get("expires_in", 3600),
        )
        sys.exit(0)
    else:
        # 刷新失败
        error_msg = result.get("error", "Unknown error")
        suggestion = "Run 'pixiv-downloader login' to re-login."
        output_refresh_result(False, error=f"{error_msg} {suggestion}")
        sys.exit(1)
```

## Open Questions

1. **Token mask 的前缀和后缀长度**
   - What we know: CONTEXT.md 示例显示 "abc...xyz" (3+3),但 status_cmd.py 使用 10+10
   - What's unclear: 应该使用哪个长度?
   - Recommendation: 使用 10+10 与 status_cmd.py 保持一致,提供足够信息同时保护隐私

2. **JSON 输出的 ensure_ascii 参数**
   - What we know: 标准 json.dumps() 默认 ensure_ascii=True,会转义非 ASCII 字符
   - What's unclear: 是否需要支持中文错误消息?
   - Recommendation: 使用 ensure_ascii=False 支持中文错误消息,提升用户体验

3. **日志级别**
   - What we know: 主命令有 --verbose 和 --quiet 选项,但 refresh 命令输出纯 JSON
   - What's unclear: 是否应该支持这些选项?如果支持,日志输出到哪里?
   - Recommendation: 不在 refresh 命令中支持 --verbose/--quiet,所有日志通过 logger 输出到 stderr,JSON 输出到 stdout,避免混合输出
