# Phase 02.1: 独立的 token 刷新命令 - Context

**Gathered:** 2026-02-24
**Status:** Ready for planning

<domain>
## Phase Boundary

添加一个独立的 CLI 命令 `pixiv-downloader refresh`,用于手动或定时刷新 Pixiv token,避免 token 过期。这个命令专注于 token 刷新功能,不涉及登录或登出操作。

</domain>

<decisions>
## Implementation Decisions

### 命令接口设计
- **命令名称:** `pixiv-downloader refresh`
- **参数:** 无必需参数,保持简单
- **可选参数:**
  - `--help/-h`: 显示帮助信息,包括所有参数功能描述和返回信息描述
- **输出格式:** 默认 JSON 格式输出 (方便第三方程序解析)
- **Token 显示:** 显示部分 token (如: abc...xyz),保护安全性的同时提供足够信息

### JSON 输出结构
```json
{
  "status": "success" | "failed",
  "expires_at": "2026-02-24 20:00:00",
  "token": "abc...xyz",
  "error": "错误描述(仅在失败时)"
}
```

**字段说明:**
- `status`: 刷新结果状态
- `expires_at`: token 过期时间
- `token`: 新 token 的部分内容(前缀...后缀)
- `error`: 失败时的错误详情

### 刷新行为
- **刷新前验证:** 总是直接刷新,不验证当前 token 有效性
- **自动保存:** 刷新成功后自动保存新 token 到加密存储
- **失败处理:** 刷新失败时明确提示用户运行 `pixiv-downloader login` 重新登录

### 错误处理
- **Token 不存在:** 提示运行 `pixiv-downloader login` 命令
- **网络错误:** 立即失败,不自动重试 (让第三方程序决定重试策略)
- **退出码:**
  - `0`: 刷新成功
  - `1`: 刷新失败 (包括 token 不存在、网络错误、刷新失败等所有错误情况)

### Claude's Discretion
- JSON 格式化的具体实现
- Token 部分显示的具体格式 (显示多少字符)
- 错误消息的详细程度和格式
- 日志记录级别

</decisions>

<specifics>
## Specific Ideas

- **第三方程序友好:** 默认 JSON 输出使得第三方程序(如定时任务脚本)可以轻松解析返回信息
- **简单直接:** 无必需参数,开箱即用
- **标准退出码:** 遵循 Unix 标准,便于脚本判断成功/失败
- **清晰的错误提示:** 失败时提供明确的解决建议 (如: 运行 login 命令)

</specifics>
