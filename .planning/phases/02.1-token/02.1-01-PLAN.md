  <action>
创建 `refresh_cmd.py` 文件,实现独立的 token 刷新命令。

**实现要点:**
1. 使用 Click 框架定义 `@click.command()` 命令函数 `refresh()`
2. 复用 `PixivOAuth.validate_refresh_token()` 刷新 token (参考 RESEARCH.md 代码示例)
3. 复用 `TokenStorage.load_token()` 和 `save_token()` 加载和保存 token
4. 实现 `mask_token()` 函数遮蔽 token (前缀10字符 + "..." + 后缀10字符)
5. 实现 `output_refresh_result()` 函数输出 JSON (使用 `json.dumps()` 而非 Rich)
6. 遵循 CONTEXT.md 定义的 JSON 输出结构:
   ```json
   {
     "status": "success" | "failed",
     "expires_at": "2026-02-24 20:00:00",  # 仅成功时
     "token": "abc...xyz",                   # 仅成功时
     "error": "错误描述"                     # 仅失败时
   }
   ```
7. 退出码: 成功返回 `sys.exit(0)`,所有错误情况返回 `sys.exit(1)`
8. 不自动重试网络错误 (让第三方程序决定重试策略)

**错误处理场景:**
- Token 不存在: 提示运行 `pixiv-downloader login`
- 网络错误: 立即失败,返回退出码 1
- 刷新失败: 提示运行 `pixiv-downloader login` 重新登录

**代码模式参考:**
- 参考 `status_cmd.py` 的 token 刷新和保存逻辑 (第 59-90 行)
- 使用 `print()` 输出 JSON,不要用 `console.print()` (避免 ANSI 转义序列)
- 使用 `ensure_ascii=False` 支持中文错误消息

**避免:**
- 不要使用 Rich Console 输出 JSON (会添加控制字符)
- 不要在 stdout 混合日志输出 (日志应输出到 stderr)
- 不要过度设计错误分类 (所有错误统一返回退出码 1)
