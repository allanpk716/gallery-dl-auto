   ```python
   class DownloadSuccessData(BaseModel):
       """下载成功时的数据结构"""
       total: int = Field(..., description="总图片数")
       success_count: int = Field(..., description="成功下载数")
       failed_count: int = Field(..., description="失败下载数")
       output_dir: str = Field(..., description="输出目录路径")
       date: str = Field(..., description="排行榜日期")
       mode: str = Field(..., description="排行榜类型")
       path_template: str | None = Field(None, description="路径模板")
       success_list: list[dict[str, Any]] = Field(default_factory=list)
       failed_list: list[dict[str, Any]] = Field(default_factory=list)
   ```

4. **DownloadOutput 模型** (顶层输出结构):
   ```python
   class DownloadOutput(BaseModel):
       """download 命令的输出结构"""
       success: bool
       data: DownloadSuccessData | None = None
       error: ErrorDetail | None = None

       def to_json(self) -> str:
           """序列化为 JSON 字符串

           使用 Pydantic 的 model_dump_json() 方法,配置:
           - exclude_none=True: 移除 None 字段 (成功时无 error,错误时无 data)
           - indent=2: 美化输出
           - ensure_ascii=False: 支持中文等 Unicode 字符
           """
           return self.model_dump_json(
               exclude_none=True,
               indent=2,
               ensure_ascii=False
           )
   ```

**关键决策**:
- 使用 Pydantic v2 的 `model_dump_json()` 方法而非 `json.dumps(model_dump())` (更简洁)
- `exclude_none=True` 避免成功响应中出现 `"error": null` (更简洁)
- 所有字符串字段使用 `str` 类型,Pydantic 会自动处理序列化
- 嵌套的字典使用 `dict[str, Any]` (success_list 和 failed_list 包含动态结构)

**参考现有模式**:

- 参考 download_cmd.py 当前的输出结构 (确保向后兼容)

- 不要在模型中添加业务逻辑 (仅数据建模)
- 不要硬编码中文错误消息 (由调用方提供)

```python
from enum import Enum

class ErrorCode(str, Enum):
    """标准化错误码

    使用 str 继承确保序列化时输出字符串而非数字。
    错误码命名规范: {模块}_{错误类型}
    """
    # 认证相关 (AUTH_*)
    AUTH_TOKEN_NOT_FOUND = "AUTH_TOKEN_NOT_FOUND"
    AUTH_TOKEN_EXPIRED = "AUTH_TOKEN_EXPIRED"
    AUTH_TOKEN_INVALID = "AUTH_TOKEN_INVALID"
    AUTH_REFRESH_FAILED = "AUTH_REFRESH_FAILED"

    # API 相关 (API_*)
    API_NETWORK_ERROR = "API_NETWORK_ERROR"
    API_RATE_LIMIT = "API_RATE_LIMIT"
    API_SERVER_ERROR = "API_SERVER_ERROR"
    API_INVALID_RESPONSE = "API_INVALID_RESPONSE"

    # 文件系统相关 (FILE_*)
    FILE_PERMISSION_ERROR = "FILE_PERMISSION_ERROR"
    FILE_DISK_FULL = "FILE_DISK_FULL"
    FILE_INVALID_PATH = "FILE_INVALID_PATH"

    # 下载相关 (DOWNLOAD_*)
    DOWNLOAD_FAILED = "DOWNLOAD_FAILED"
    DOWNLOAD_TIMEOUT = "DOWNLOAD_TIMEOUT"

    # 参数相关 (INVALID_*)
    INVALID_ARGUMENT = "INVALID_ARGUMENT"
    INVALID_DATE_FORMAT = "INVALID_DATE_FORMAT"

    # 内部错误 (INTERNAL_*)
    INTERNAL_ERROR = "INTERNAL_ERROR"
```

**关键决策**:
- 继承 `str` 和 `Enum` 确保序列化时输出字符串 (如 "AUTH_TOKEN_NOT_FOUND" 而非 1)
- 错误码按模块分组 (AUTH_, API_, FILE_, DOWNLOAD_, INVALID_, INTERNAL_)
- 使用描述性名称,避免缩写 (清晰易懂)
- 覆盖当前项目中所有已知的错误场景 (基于现有 CLI 命令分析)

**参考现有模式**:
- 参考 Phase 02.1 的 refresh 命令中使用的错误消息
- 参考 Phase 03 的 download 命令中的错误处理

**不要做**:
- 不要添加错误消息 (错误码仅是标识符,消息由调用方提供)
- 不要使用数字枚举值 (必须继承 str)
- 不要过度细分错误码 (够用即可,避免过度工程)
