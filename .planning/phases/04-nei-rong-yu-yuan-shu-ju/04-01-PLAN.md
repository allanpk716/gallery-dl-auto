实现路径模板系统,支持变量替换和路径清理

**实现步骤**:

1. **创建 src/gallery_dl_auo/utils/path_template.py**:
   参考 RESEARCH.md 的 Pattern 2 实现:

   - `class PathTemplate`:
     - `VARIABLES` 类属性: 支持的模板变量集合 `{"mode", "date", "illust_id", "title", "author", "author_id"}`
     - `__init__(self, template: str)`: 初始化模板字符串
     - `render(self, context: dict) -> Path`: 渲染路径模板
       1. 遍历 VARIABLES,替换 `{var}` 为 `context.get(var, "unknown")`
       2. 使用 `sanitize_filepath(path_str)` 清理非法字符
       3. 返回 Path 对象

   - 添加类型注解和文档字符串
   - 使用 from pathvalidate import sanitize_filepath
   - 使用 from pathlib import Path

2. **创建 tests/utils/test_path_template.py**:
   参考 RESEARCH.md 的 Validation Architecture 部分编写测试:

   - `test_template_render_basic()`: 测试基本变量替换
     - 模板: `"{mode}/{date}/{title}.jpg"`
     - 上下文: `{"mode": "daily", "date": "2026-02-25", "title": "Sunset"}`
     - 预期: `Path("daily/2026-02-25/Sunset.jpg")`

   - `test_template_render_all_variables()`: 测试所有变量
     - 模板包含所有 VARIABLES
     - 验证所有变量都被替换

   - `test_template_missing_variable()`: 测试缺失变量替换为 "unknown"
     - 模板: `"{mode}/{author}/{title}.jpg"`
     - 上下文不包含 `author`
     - 预期路径包含 "unknown"

   - `test_template_sanitization()`: 测试路径清理
     - 模板包含非法字符: `"{mode}/{author}/{title}.jpg"`
     - 上下文: `{"author": "Artist: Name", "title": "Title? With*Chars"}`
     - 预期: Windows 上的非法字符 `:`, `?`, `*` 被清理

   - `test_template_with_author_id()`: 测试 author_id 变量
     - 模板: `"{author_id}/{title}.jpg"`
     - 验证数字转换为字符串

**关键点**:
- 使用简单的字符串替换,不要引入 Jinja2 等复杂模板引擎
- 所有路径必须通过 `sanitize_filepath()` 清理
- 缺失变量统一替换为 "unknown"
- 返回 Path 对象而非字符串
