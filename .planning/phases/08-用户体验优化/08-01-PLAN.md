创建 ProgressReporter 类,封装详细模式下的进度显示逻辑。

实现要求:
1. __init__(self, verbose: bool = False)
   - verbose: 是否启用详细模式
   - console: Console(stderr=True) 实例(避免污染 stdout)

2. 方法:
   - update_progress(idx: int, total: int, failed: int): 更新进度状态
     * 详细模式:输出格式 `[2026-02-25 14:23:15] 下载中: 24/30 (失败: 2)`
     * 静默模式:无输出
     * 时间戳使用 datetime.now().strftime("%Y-%m-%d %H:%M:%S")

   - report_success(title: str, illust_id: int): 报告成功下载
     * 详细模式:输出 `  ✓ {title} (ID: {illust_id})`,绿色样式
     * 静默模式:无输出

   - report_rate_limit_wait(delay: float): 报告速率控制等待
     * 详细模式:输出 `  等待 {delay}s...`,dim 样式
     * 静默模式:无输出

   - report_retry(attempt: int, max_retries: int, error: str): 报告重试
     * 详细模式:输出 `  重试 {attempt}/{max_retries}: {error}`,黄色样式
     * 静默模式:无输出

3. 所有方法在 verbose=False 时立即返回,无任何输出

4. 使用 Rich Console 的 style 参数实现颜色输出:
   - 绿色(style="green"):成功
   - 黄色(style="yellow"):重试、等待
   - dim(style="dim"):辅助信息

实现细节:
- 不要使用 print(),始终使用 console.print()
- Console 初始化为 stderr=True,确保不污染 stdout 的 JSON 输出
- 时间戳使用本地时间,格式 YYYY-MM-DD HH:MM:SS
- 不实现进度条可视化(已 deferred)

测试策略:
- 测试详细模式输出到 stderr
- 测试静默模式无输出
- 测试时间戳格式
- 测试样式应用(Rich markup)
