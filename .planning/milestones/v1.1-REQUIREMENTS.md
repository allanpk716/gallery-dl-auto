# Requirements: Pixiv 排行榜下载器 (gallery-dl-auto)

**Defined:** 2026-02-24
**Last Updated:** 2026-02-25 after Phase 8 completion
**Core Value:** 自动化获取 Pixiv refresh token 并下载排行榜内容 — 用户首次手动登录后,程序自动捕获、存储和更新 refresh token,无需手动从浏览器开发者工具中复制,实现真正的自动化下载流程

## v1 Requirements

初始发布的需求。每个映射到路线图阶段。

### 认证与 Token 管理

- [x] **AUTH-01**: 用户能够通过自动化流程获取 pixiv 的 refresh token(首次需手动登录)
- [x] **AUTH-02**: 程序自动保存 refresh token 到安全存储
- [x] **AUTH-03**: 程序自动验证 refresh token 是否有效
- [x] **AUTH-04**: 程序自动刷新过期的 refresh token

### 排行榜下载

- [x] **RANK-01**: 用户能够下载 pixiv 每日排行榜
- [x] **RANK-02**: 用户能够下载 pixiv 每周排行榜
- [x] **RANK-03**: 用户能够下载 pixiv 每月排行榜
- [x] **RANK-04**: 用户能够通过参数指定要下载的排行榜类型

### 内容下载

- [x] **CONT-01**: 程序下载排行榜中的图片文件
- [x] **CONT-02**: 程序获取作品的基础元数据(标题、作者、标签)
- [x] **CONT-03**: 程序获取作品的扩展统计数据(收藏数、浏览量、评论数等)
- [x] **CONT-04**: 用户能够指定图片下载的保存路径

### 输出与接口

- [x] **OUTP-01**: 程序以 JSON 格式返回下载结果汇总(成功数、失败数、总数)
- [x] **OUTP-02**: 程序以 JSON 格式返回每张图片的详细信息(URL、标题、作者、标签、统计数据)
- [x] **OUTP-03**: 程序以 JSON 格式返回下载文件的路径
- [x] **OUTP-04**: 程序以 JSON 格式返回错误信息和失败原因
- [x] **OUTP-05**: 用户能够在终端通过命令行参数调用程序
- [x] **OUTP-06**: 程序提供清晰的命令行帮助信息和参数说明

### 用户体验

- [x] **UX-01**: 程序处理网络错误并提供清晰的错误提示
- [x] **UX-02**: 程序处理权限错误并提供清晰的错误提示
- [x] **UX-03**: 程序支持增量下载,跳过已下载的内容
- [x] **UX-04**: 程序实时显示下载进度(进度条或状态信息)
- [x] **UX-05**: 程序控制下载速率以避免触发 Pixiv 的反爬虫机制
- [x] **UX-06**: 用户能够配置请求间隔和并发数

## Out of Scope

明确排除的功能。记录以防止范围蔓延。

| Feature | Reason |
|---------|--------|
| GUI 界面 | 专注于 CLI + JSON 输出,让用户构建自己的 UI — 偏离 CLI 工具定位,限制自动化集成能力 |
| 批量账号管理 | 单账号设计 — 增加复杂度,违反 pixiv 使用条款风险,单账号已满足大部分场景 |
| 自动上传/分享 | 专注于下载 — 版权问题,平台政策风险,功能蔓延 |
| 图片格式转换/编辑 | 保持原图质量 — 偏离核心功能,依赖复杂,性能开销 |
| 下载非排行榜内容 | 专注排行榜场景 — 功能蔓延,复杂度增加,维护负担重(搜索结果、用户作品集等) |
| 代理/VPN 集成 | 用户自行配置 — 网络环境复杂多样,难以通用化,增加维护成本(让用户自行配置系统代理环境变量) |
