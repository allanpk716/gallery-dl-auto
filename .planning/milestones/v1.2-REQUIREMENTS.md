# Requirements: gallery-dl-auto v1.2

**Defined:** 2026-02-25
**Core Value:** 自动化获取 Pixiv refresh token 并下载排行榜内容 — 用户首次手动登录后,程序自动捕获、存储和更新 refresh token,无需手动从浏览器开发者工具中复制,实现真正的自动化下载流程

## v1.2 Requirements

第三方 CLI 集成优化 — 确保工具完全适配作为第三方工具的 CLI 依赖。

### CLI API 接口

- [x] **CLI-01**: 用户可以通过 --json-help 参数获取所有命令的结构化 JSON 帮助信息(包含命令名称、参数、描述等元数据)
- [x] **CLI-02**: 用户可以通过 --quiet 参数禁用所有进度和日志输出,仅返回最终 JSON 结果(适合第三方工具调用)
- [x] **CLI-03**: 用户通过 --json-output 参数确保所有输出(包括错误信息)都是有效的 JSON 格式(保证可解析性)

### 文档

- [x] **DOCS-01**: 开发者可以查阅 INTEGRATION.md 文档了解如何作为第三方工具集成(包含调用方式、参数说明、最佳实践)
- [x] **DOCS-02**: 开发者可以参考 Python 和命令行调用示例代码(包含常见场景的完整示例)
- [x] **DOCS-03**: 开发者可以查阅完整的退出码文档,了解每个退出码的含义和使用场景(便于自动化流程判断执行状态)

### 验证

- [x] **VAL-01**: 所有现有命令的 JSON 输出格式经过验证,符合规范(确保第三方工具能正确解析)
- [x] **VAL-02**: 所有退出码经过验证,符合文档说明(确保自动化流程能准确判断执行状态)
- [x] **VAL-03**: 第三方工具调用场景经过集成测试验证(确保实际使用场景的可靠性)

## Out of Scope

明确排除的功能,避免范围蔓延。

| Feature | Reason |
|---------|--------|
| 进度条可定制化 | 保持简单 --quiet 即可,让第三方工具决定如何显示进度 |
| API 版本控制 | v1.2 专注于验证现有 API,版本控制在后续里程碑考虑 |
| Python SDK 封装 | 当前保持 CLI + JSON 输出模式,SDK 封装需要更多设计考量 |
| 配置文件自动生成 | 集成文档中提供示例即可,不需要自动化工具 |
