**Goal**: 优化 CLI 用户体验,提供详细模式和速率控制配置
**Depends on**: Phase 7 (生产级别错误处理基础)

  4. 用户能通过 CLI 参数覆盖速率控制设置
**Plans**: 2 plans

Plans:

#### Phase 8.1: CLI API 增强
**Goal**: 提供完整的第三方 CLI 集成 API 接口
**Depends on**: Phase 8 (用户体验优化基础)
**Requirements**: CLI-01, CLI-02, CLI-03
**Success Criteria** (what must be TRUE):
  1. 用户可以通过 --json-help 参数获取结构化的命令元数据(包含所有命令名称、参数、描述)
  2. 用户使用 --quiet 参数时看不到任何进度和日志输出,仅收到最终 JSON 结果
  3. 用户使用 --json-output 参数时所有输出都是有效 JSON,包括错误信息也能被 JSON 解析器正确解析
**Plans**: 3 plans

Plans:
- [ ] 08.1-01: JSON 帮助系统 — 实现 --json-help 全局参数和结构化命令元数据
- [ ] 08.1-02: 静默和 JSON 输出模式 — 实现 --quiet 和 --json-output 全局参数
- [ ] 08.1-03: 集成验证 — 端到端测试验证所有命令的 CLI API 行为
