## Tech Debt

### Phase 10: API Validation

**Item 1: Windows Encoding in Integration Tests**
- **Tests affected:** test_subprocess_status_command, test_subprocess_config_command
- **Root cause:** Windows subprocess with Rich table output uses CP1252 encoding, tests expect UTF-8
- **Severity:** LOW - Does NOT affect JSON output functionality
- **Workaround:** Third-party tools should use `--json-output` mode (recommended in INTEGRATION.md)
- **Recommendation:** Update tests to use `--json-output` mode instead of parsing Rich tables

**Item 2: Documentation Tracking Lag**
- **Issue:** REQUIREMENTS.md shows CLI-01/02/03 as "Pending" but they are complete
- **Root cause:** REQUIREMENTS.md not updated after Phase 8.1 completion
- **Severity:** LOW - Documentation-only issue
- **Recommendation:** Update REQUIREMENTS.md to mark CLI-01/02/03 as Complete and assign to Phase 8.1

### Phase 08.1: CLI API Enhancement

**Item 1: Command-Specific JSON Output Incomplete**
- **Issue:** Some commands (status, config) still output Rich tables in default mode
- **Root cause:** Phase 8.1 focused on global parameter infrastructure, command-specific JSON logic deferred
- **Severity:** LOW - `--json-output` mode works correctly
- **Impact:** None - Third-party tools should use `--json-output` mode as documented in INTEGRATION.md
- **Recommendation:** Future enhancement to implement JSON output for all commands in all modes
