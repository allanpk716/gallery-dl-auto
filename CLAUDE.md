# CLAUDE.md - 项目开发规范

本文档定义了 gallery-dl-auto 项目的开发规范和 Claude Code 工作偏好。

## 语言和沟通

- **使用中文回答问题**：所有与用户的沟通、解释和文档均使用中文

## 开发环境

- **操作系统**：Windows 开发系统
- 注意 Windows 特定的路径分隔符（`\`）和换行符（`\r\n`）

## 编码规范

### BAT 脚本开发

- **BAT 脚本中不要包含中文字符**：避免编码问题和兼容性问题

### 测试代码管理

- **临时测试代码和数据**：统一放置在项目根目录的 `tmp` 文件夹中
- 保持项目结构整洁，避免临时文件散落在各处

## 脚本维护

- **脚本修复原则**：在原有脚本基础上进行修复，非必要不创建新脚本
- 避免产生重复或功能相似的脚本文件

## MCP 和 Agent 使用

### 图片识别服务

- **图片尺寸限制**：在使用截图的 MCP 或 Agent 能力前，确保图片尺寸小于 1000x1000
- 在提交到识别图片服务之前进行必要的图片压缩或裁剪

## 项目文档结构

### 计划文档

- **项目计划文件**：放置在 `docs\plans` 目录中

### Bug 跟踪

- **自测发现的 bug**：记录在 `docs\bugs` 目录中

## 测试策略

### 前后端分离项目

- **浏览器测试**：使用 dev-browser 技能进行前后端通信和交互操作的测试
- 确保真实环境下的集成测试，而非仅依赖单元测试
