# 需求：排行榜下载跨日去重

## 1. 背景

Pixiv 排行榜每天更新，但热门作品经常连续多日上榜。例如：
- 作品 A 在 3月7日 日榜排名第 5
- 同一作品 A 在 3月8日 日榜排名第 12

当前系统会将作品 A 下载两次，分别保存到不同日期目录。

## 2. 问题描述

### 当前行为

```
pixiv-downloads/
└── pixiv/
    └── rankings/
        └── day/
            ├── 2026-03-07/
            │   └── 12345_p0.jpg  ← 第1次下载
            └── 2026-03-08/
                └── 12345_p0.jpg  ← 第2次下载（重复）
```

### 痛点

1. **浪费带宽**：同一图片重复下载多次
2. **浪费存储**：虽然文件在不同目录，但内容相同
3. **用户体验差**：用户浏览时看到重复图片

### 影响范围

- 日榜：受影响最大，热门作品可能连续 3-7 天上榜
- 周榜：受影响较小，但月末/月初可能有重叠
- 月榜：基本不受影响

## 3. 需求目标

### 核心目标

**同一作品 (illust_id) 只下载一次，无论出现在多少天的排行榜中。**

### 期望行为

```
pixiv-downloads/
└── pixiv/
    └── rankings/
        └── day/
            ├── 2026-03-07/
            │   └── 12345_p0.jpg  ← 首次下载
            └── 2026-03-08/
                └── (12345 跳过，不重复下载)
```

## 4. 解决方案

### 方案 A：SQLite 统一去重（推荐）

利用现有 `DownloadTracker` 组件：

1. **gallery-dl 引擎集成 tracker**
   - 下载前查询 `is_downloaded(illust_id)`
   - 已下载则跳过，记录日志
   - 下载成功后调用 `mark_downloaded(illust_id)`

2. **优点**
   - 复用现有代码，改动小
   - 支持跨引擎统一去重
   - 数据库可查询历史记录

3. **缺点**
   - 需要维护 SQLite 数据库

### 方案 B：文件系统去重

下载前检查所有日期目录是否已存在该文件：

1. **实现方式**
   ```python
   def is_file_exists(illust_id: int, base_dir: Path) -> bool:
       pattern = f"{illust_id}_p*.*"
       for date_dir in base_dir.glob("rankings/day/*"):
           if list(date_dir.glob(pattern)):
               return True
       return False
   ```

2. **优点**
   - 无需数据库
   - 直观易懂

3. **缺点**
   - 文件多时性能差（需要扫描大量目录）
   - 如果用户手动删除文件，数据库状态不一致

### 方案 C：硬链接去重

下载后使用硬链接，多日目录共享同一物理文件：

1. **实现方式**
   - 所有图片下载到 `pool/` 目录
   - 日期目录创建硬链接指向 `pool/` 中的文件

2. **优点**
   - 节省存储空间
   - 保留原有目录结构

3. **缺点**
   - 实现复杂
   - 跨文件系统不支持
   - Windows 权限要求高

## 5. 推荐方案

**采用方案 A：SQLite 统一去重**

### 理由

1. `DownloadTracker` 已实现并测试通过
2. 改动量最小，只需修改 `gallery_dl_wrapper.py`
3. 支持 `--skip-history` 参数让用户选择是否跳过

### 改动点

| 文件 | 改动内容 |
|------|----------|
| `download_cmd.py` | 初始化 `DownloadTracker` 并传递给 wrapper |
| `gallery_dl_wrapper.py` | 下载前检查 tracker，下载后更新 tracker |

### 预估工作量

- 代码改动：约 30 行
- 测试：约 1 小时

## 6. 验收标准

1. ✅ 下载 3月7日 日榜后，3月8日 日榜中重复作品不再下载
2. ✅ 日志中显示 `[SKIP]` 记录被跳过的作品
3. ✅ 统计数据准确：`skipped` 计数正确
4. ✅ 支持 `--no-skip` 参数强制重新下载

## 7. 后续优化（可选）

1. **去重统计报告**：显示本次下载跳过了多少重复作品
2. **清理重复文件**：提供命令清理已存在的重复文件
3. **白名单机制**：某些用户可能希望保留每日独立副本
